    • Translations for Firewall Services Defined by Packages

    Description

    SuSEfirewall2 has been supporting FW services defined by packages for some time. YaST Firewall currently uses a workaround that make their name and description translatable. This functionality is now obsolete and will slowly die. Why? It needs to be done manually in YaST sources. Before creating the final translations, someone has to run script refresh-srv-def-by-pkgs-trans.sh in http://svn.opensuse.org/svn/yast/trunk/yast2/library/network/src/ This will recreate yast2-services-translations.ycp in the same location. Then someone needs to create pot files and submit them. In ideal world this would be automatic.

    Moreover, the scrip mentioned above uses pdb command which is internal to SUSE and the database is outdated as it's not supported for openSUSE 11.2 ad further.
