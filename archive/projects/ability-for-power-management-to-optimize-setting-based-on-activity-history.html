    • Ability for power management to optimize setting based on activity history

    Tags: Desktop, Mobile, LowPower, SLED, Idea

    Description

    Use a history of system activity to learn when to put system to sleep rather than just using a set period of time. For example:

    * between 9:00 - 5:00 if keyboard is inactive for more than 10 minutes after the start of a new hour (1:00, 2:00, etc.) then system has 95% chance to be inactive for an hour (person at meeting)

    * If idle for more that 10 minutes after 5:00 PM then system will not be active until 9:00 AM next work day.

    Tim Lee originated this idea. No one is currently working on implementing this idea (it's up for grabs!).
