    • Add gcc profile feedback support to some packages

    Tags: Idea, performance

    Description

    gcc can improve the performance of programs by using profile feedback. Compile the program with -fprofile-generate, run the program with some training load, recompile with -fprofile-use. In some cases this can give very noticeable speedups.

    A few rpms (like sed or gcc itself) use this already, but not enough. For anything that already has a test suite it should be reasonable easy to add to the Makefiles. Just run the test suite once generating profile data and then recompile the program again using the feedback data. Do this for a few packages.

    A nice way might be also to add support for this for popular "meta" Makefile generators like CMake or automake. Then it might be easier to add this to a whole class of programs.

    People

    Andi Kleen originated this idea. No one is currently working on implementing this idea (it's up for grabs!).
