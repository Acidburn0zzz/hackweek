    • Add more encryption / signing client / server tools to yast

    Tags: Idea, Security, openSUSE, SLES, Management

    Description

    The world is getting more and more dependent on confirmation of identities and managing of them. So would it not be a good idea to centralize a the basics in yast so also non technical people can use them.

    What should be implemented is creating/deleting (gpg) keys. Uploading or creating of keyservers.

    Also other types of signing and encrypting should / could be added. Both server as client side.

    Options:

    • gpg client / server
    • openID

    People

    Mike Veltman originated this idea. No one is currently working on implementing this idea (it's up for grabs!).
