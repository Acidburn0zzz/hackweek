    • Anti-RSI keyboard configuration

    Tags: Idea, openSUSE, SLES, SLED, Desktop, X11, keyboard, RSI

    Description

    I read somewhere that to avoid RSI it is recommended to enter key combinations using both hands, instead of one hand with stretched-out fingers pressing both keys. Having typed for some 25 years now, I find it absolutely impossible to heed that advice. The solution may be to make the system enforce this behaviour: Create a keyboard configuration that disables all key combinations with Shift/Alt/Ctrl keys that can be reached with one hand, forcing the user to use the other Shift/Alt/Ctrl key with their other hand. For instance, disable LeftShift + 2 and RightCtrl + O so the user needs to use RightShift + 2 and LeftCtrl + O, which is impossible to do with one hand.

    People

    Ulrich Hecht originated this idea. No one is currently working on implementing this idea (it's up for grabs!).
