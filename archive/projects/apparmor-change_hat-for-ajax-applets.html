    • AppArmor change_hat for AJAX Applets

    Tags: Idea, Security, Desktop, AppArmor, Web20, ajax

    Description

    AppArmor has a change_hat() API that lets a process tell the kernel that it will be working on a different security context for a while. This is intended for server platforms, e.g. Apache mod_perl, where an Apache module tells the kernel that Apache is about to start interpreting a PERL script, so that you can have separate "hats" for each servlet you host, such as a PERL script or a PHP page, even though they never appear in the kernel process table. change_hat modules exist for Apache, Java Tomcat 4 and 5, and PAM.

    Web applets are little programs that run in your web browser. In the hip new world of Web 2.0, everything is AJAX, and the client side of that is Javascript applets. Unfortunately Javascript applets have crappy security. Fortunately, the Firefox web browser is open source, and does almost everything in a high level language (Xul).

    The idea: steal the Apache mod_apparmor code and add it to Firefox, so that there is an AppArmor "hat" wrapped around the crappy Javascript applets that the web makes you run if you, e.g. want to log in to Innerweb.

    People

    Crispin Cowan originated this idea. No one is currently working on implementing this idea (it's up for grabs!).
