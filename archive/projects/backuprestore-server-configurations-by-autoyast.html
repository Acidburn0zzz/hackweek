    • 2 - Command line support for autoyast

    Tags: Idea, YaST, Management, Network, UnixFeature, Server, LAMP

    Description

    At the end of installation there is checkbox for create autoyast profile. Also we can start Autoyast module, configure from default and export profile.

    But there is no way how to create profile for particular services on running system. Read and Write buttons in Autoyast module can do that.

    Example why its cool ;-):

    We have configured dhcp server running on machine A and want to migrate to machine B. Start autoyast on machine A, select dhcp service, click Read in menu and Export. Now start autoyast on machine B, Import profile, click Write.

    This could be usefull to create backup (or many) for any service configurable by YaST and restore from it.

    Example:

    autoyast http-server export=/var/lib/autoinstall/repository/module.xml

    In case you want restore from profile :

    autoyast http-server import=/var/lib/autoinstall/repository/module.xml

    Module list for work with several modules :

    autoyast modules=[http-server print-server] export=/tmp/profilename.xml
