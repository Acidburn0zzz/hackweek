    • Create a repository for sharing Mono libraries

    Tags: Idea, Mono, DevTool

    Description

    The idea is to create a software infrastructure for publishing and sharing Mono libraries. Something like Perl's CPAN, but for Mono.

    It would not be just an online repository of libraries, but it would provide a set of client tools for locating, downloading and installing those libraries.

    Some considerations:

    * The main goal would to share fully managed libraries. * It would be a development only tool. Downloaded packages should be only for development, not for distribution. * It should support sharing libraries with or without the source code. * Libraries might depend on other libraries. * Libraries might include monodoc documentation.

    What should it provide:

    * A library for accessing to the repository, downloading and installing libraries. * A tool for doing the previous from the command line. * A MonoDevelop add-in for doing the same.

    People

    Lluis Sanchez originated this idea. No one is currently working on implementing this idea (it's up for grabs!).
