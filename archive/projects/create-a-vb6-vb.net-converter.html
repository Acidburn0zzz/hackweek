    • Create a VB6 -> VB.Net converter

    Tags: Idea, DevTool, Mono

    Description

    The VB6 -> VB.Net converter currently available isn't really an option for anybody that has a decent sized VB6 project, because of a few issues:

    * It's not possible to tweak or change the output, the converter has decided it has one way to do it that's it. And it's very easy to see that it's been machine-generated.

    * The converted code is tweaked for maximium compatibility, it's not maintainable, and it's certainly not easy to make it look .Netish.

    * It generates a huge amount of todos, warnings, and quite a few errors as well, that has to be fixed before anything can be done. And with a medium-size project the same 10 error causes would show up 100 times, this can be quite a task (and very error-prone).

    A better solution would solve these problems, by offering options for the generated output:

    * Allow redirection from a VB6 dll/class/method to another (maybe completely) different .Net assembly/class/method, possibly by using xml files. With this it's also possible to set up different profiles, i.e. a --max-compat option, or a --max-netish, or whatever.

    * Make as much as possible of the code-generation configurable.

    People

    Rolf Bjarne Kvinge originated this idea. No one is currently working on implementing this idea (it's up for grabs!).
