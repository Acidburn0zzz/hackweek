    • Create non-root system

    Tags: Idea, Desktop, Security, System

    Description

    It would be nice if a user could administer their system without having to know about root.

    We have a lot of tools like sudo, cups policies, PolicyKit, DBus, but no UI and nothing putting it all together. We are part way there with the 10.3 %wheel setup allowing a user to become root with their own password if they are in the while group. We could tag a user or users on the system as "admin" users like MacOS X or Windows and allow those users to have more power. Perhaps guarding really important operations by requesting the users password again (like MacOS X for a system update).

    Klaus Kampf calls this "role based yast" so the Yast may have further background.

    People

    JP Rosevear originated this idea. No one is currently working on implementing this idea (it's up for grabs!).
