    • Cross-desktop unified UI -- Defining the "native" open source look and feel

    Tags: KDE, GNOME, Desktop, InProgress

    Description

    All desktop applications should: * Have the same look and feel * Use the same icons * Be designed with a consistent with a set of guidelines in mind

    ...Of course, this also means that there should be a set of guidelines.

    Currently, we have several toolkits; it would be nice to have each of them use a UI graphics library that they could call upon to draw buttons, scrollbars, etc. Obviously, most people are thinking of both Qt and GTK+ at this point, but we actually use so many other toolkits currently, as well... Think about Mozilla's XUL, Mono's implementation of Windows.Forms, Wine, Java SWING, and OOo's own widgets, for instance.

    Right now, we do have a little bit of rubber meeting the road in the area of icons. [[http://tango.freedesktop.org/Tango_Desktop_Project|The Tango Desktop Project]] has made a cross-desktop standard for icon naming and provides a reference set of excellent icons which can be used in any software.

    However, when it comes to widgets, people have implemented themes which mimic either GTK+ or Qt, and do so poorly. Theming on Linux is currently quite a broken mess and implemented in a shoddy way wherever themes are used.

    Implementation suggestions to realize this idea: * Work on a common, shared UI suggestions document based on KDE & GNOME HIGs / style guides * Develop a library and theme format to specify the way certain widgets should be drawn; this should take advantage of all the nice rendering stuff we now have in the desktop (cairo, transparency, SVG, etc.) -- in addition to just the look of widgets themselves, it should also contain some information about policy, such as various suggested padding sizes (internal and external to widgets, such as padding and margin in CSS), and should include spacing with different meanings in mind as well (associated widgets should be spaced differently from one another than widgets not associated) * Make a theme format which is based on CSS, so that certain widgets can be styled, with cascades as well -- this does not need to map 1:1 to CSS used in HTML, but should be the same structure and idea -- it should also allow for multiple classes, so that a toolkit or application could request an object of a button class, and it would be drawn as a button (and inherit some basic button states, such as a hover and clicked); also, if a widget has multiple classes specified, then it should inherit the properties of both defined classes

    People

    Garrett LeSage originated this idea.

    [[http://tango.freedesktop.org/Tango_Desktop_Project|The Tango Desktop Project]] has been hard at work at making this happen in the realm of icons.

    We need excellent hackers to make this a reality for the rest of the desktop.
