    • Cross platform hardware synthesizer patch librarian & editor

    Tags: Desktop, InProgress, HelpWanted, HW-Hacker

    Description

    Digital hardware [[http://en.wikipedia.org/wiki/Synthesizer#The_modern_digital_synthesizer|synthesizers]] (which encompasses just about every synthesizer manufactured in the last 20 years) store their sounds as **patches** in a **patch bank**. Synthesizers have a limited amount of space to store patches in; a Roland XP-50, for instance, only has 128 patch slots in their user-modifiable patch bank, which is where the **patch librarian** comes in. The patch librarian is a piece of software running on a computer that takes a MIDI dump (or on very new synthesizers, a USB dump) of the patch bank, stores it to disk, and then can transmit individual patches, or an entire bank, back to the synthesizer whenever the user wants to, getting around that constraint.

    A **patch editor** is software that allows a user to modify the patches using the computer, rather than fiddling with knobs or navigating menu structures on the synthesizer itself. This allows for a greater degree of control over the sound, and makes modifying patches much easier.

    I've only been able to find one patch librarian/editor that works on Linux, JSynthLib, which doesn't have a very good user interface, doesn't support many synthesizers, and doesn't appear to be very easy to extend. All of the others exist solely in the commercial Windows/Mac worlds, so I think it's time to give the world good open source, cross-platform patch librarian/editor that can be quickly and easily extended to support any MIDI device. I'll be writing it in C++ using wxWidgets.
