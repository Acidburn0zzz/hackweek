    • design new cross-desktop thumbnailing system

    Tags: Idea, Desktop, Graphics, openSUSE, SLED, HelpWanted

    Description

    Current thumbnailing uses data-oriented specification using ~/.thumbnails directory.

    It seems to have many drawbacks:

    - No support for image or directory moves and (sym)linking (after move or linking, all thumbnails are generated again).

    - No support for removable media (thumbnail uses different hash if medium is insterted in drive 1 or drive 2 and are never deleted).

    - No support for large collections (defines directories, which may contain more than 100000 files),

    - Uses png (with current processors it takes more time to read 25kB PNG than read and decode 5kB JPG).

    - No support for files with embedded thumbnails (thumbnails are generated again).

    - No requirements for garbage collection.

    New system should solve all these problems, probably with a cross-desktop library.

    People

    Stanislav Brabec originated this idea. No one is currently working on implementing this idea (it's up for grabs!).
