    • enhance virt-manager to manage virtual disks (and via yast and command line)

    Tags: Idea, Virtualization, InProgress

    Description

    Too busy hacking; forgot to enter this.

    Xen's xend only manages virtual machines; it does not handle virtual storage. For example, if you delete the VM, the virtual disk image is forgotten (even though it's still taking up disk space). This confuses users. Virt-manager (since it ultimately is built on xend) suffers the same problem.

    We need a tool to list disks, and show which VM(s) (if any) are currently using them. If none are using a disk, the user can delete the disk.

    Note that this is only for on-box virtualization; this is not a datacenter tool/solution.

    Currently we have a command line tool (which shares python classes with vm-install), and the beginnings of a YaST tool, and ideas about integration into virt-manager.

    People

    Charles Coffing originated this idea, and is working on it. Mike Fritch is working on the YaST interface.
