    • Evolution support for TNEF Outlook mails with "winmail.dat" attachements

    Tags: Desktop, Evolution, Done, JudgeMe

    Description

    A lot of Evolution on Linux users receive E-Mails from Outlook users with "winmail.dat" attachements. These are roughly speaking just proprietary archives which actually contain data which Evolution is able to handle such as meeting invitations, iCal, further attachments and more.

    It would increase the value of migrating to a Linux environment if Evolution would support "unrolling" those files automatically without regular user joe to have to use command line libraries to extract the content or even worse being stuck with such a .dat file.

    Infact there is already a very [[http://www.go-evolution.org/Tnef|hackish Evolution Plugin]] which uses the [[http://ytnef.sourceforge.net/|ytnef library]] to do this job and it offers a great user experience as all the attachements are nicely shown in Evolutions preview pane even with interactive meeting invitation functionality. (Take it as a PoC)

    It would an idea, as noted by the ytnef developer himself, to create a clean plugin without external dependencies (includes tnef code) which integrates nicely into Evolution and hopefully is bundled upstream. Might be a good idea for LOADS of lizards ;)
