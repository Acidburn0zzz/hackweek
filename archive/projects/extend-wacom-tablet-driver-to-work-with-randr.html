    • Extend Wacom Tablet driver to work with RandR

    Tags: InProgress, Desktop, Mobile, openSUSE, SLED, X11

    Description

    Currently the X.org driver for wacom tablet input devices don't work with the XRandR extension. Because of this there is no automatic rotation of the related input devices if the user change the screen orientation e.g. via xrandr (or the desktop tools). The user have to change the orientation for each of the three (or more) input devices separately via a tool or via change the X.org config and restart the X server.

    The automatic on-fly rotation is very important for TabletPCs. At the moment we have a workaround for KDE in KRandRTray which call xsetwacom to rotate the wacom input devices. Unfortunatly this don't fix GNOME or all the other command line tools. The wacom driver should do this automatically without any hack in other tools.
