    • Faster OpenOffice.org Build

    Tags: Idea, Desktop, DevTool, office, HelpWanted

    Description

    OpenOffice.org is huge piece of code. It takes many hours to get the full build on normal machines. There are lots possibilities how to make life easier:

    • Most developers are interested only into a particular module, for example into OOo Writer. They do not want to build the whole application then. It would help to split the build into more pieces, create the respective devel packages...
    • Jam build tool handle dependencies much faster than dmake. Someone has already added support for jam into the Calc module. It would be fine to add it for more modules.
    • The tool localize is very slow, especially when it extracts the existing translations.
    • The SDK installation is incredibly slow. I wonder what happens there.
    • The helpcontent build takes ages and needs a lot of memory. I wonder if it could be parallelized even using icecream.

    People

    Petr Mladek originated this idea. He improved the speed of the SDK installation, see i#78892. He is going to work on the package splitting during his normal work days (after the end of hack week).
