    • Faster packager

    Tags: Idea, devtool

    Description

    IÂ´d like to help packager to do some routine procedures automatically. I'll prepare a set of scripts and wrappers that will make life of packager a bit easier. Here are some examples:

    __1) Updating the package__

    # update-to 1.2.1 - all patches will be renamed to version 1.2.1, the version in spec file will be also updated

    # add-patch new.patch - lines with new patch will be written into spec file ("Patch1: new.patch", "%patch1")

    # remove-patch obsoleted.patch - analogically

    __2) Building the package__

    BUILD_ROOT and BUILD_DIST will be set accordingly to [target] section in .setup file

    __3) Coloful logs__

    The output from abuild, Re, Rd should be colorful for better readability

    __4) Macros for Vi__

    When editing spec file I often do: jump from one section to another (%prep, %build, ...)

    jump from "Patch26: %{name}-%{version}-shell.patch" to "%patch26" i.e. to comment out the patch

    __5) Tools in BUILD_ROOT__

    I'd like to have additional packages installed into BUILD_ROOT: vi, cscope, ...

    I think when I'll go through the packagers routine tasks I find more possible improvements.
