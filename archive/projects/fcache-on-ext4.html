    • fcache on ext4

    Tags: Idea, Kernel, filesystem

    Description

    improved fcache on ext4 file system fcache is an idea brought out by Jens Axboe. Current fcache is implemented on ext3. There are several stuffs to be improved: 1) Impelment fcache on ext4 file system. 2) Current fcache is block based, not file based. Once cached block of files are modified, there is no synchronization. * implement file based fcache. Once file is modified, invoke a deamon to perform automatical fcache synchronization. * Integrated with Yast to make the configuration and running of fcache more easy for end user. 3) better grain size can be used to improve parallel to read cached block into memory. * Some cached blocks can be read into memory after kernel initiated and before init scripts runs, e.g. blocks of .so files, init scripts and so on. * Some cached blocks can be read int memory by a kernel thread while init scripts are running, e.g. execution and .so files of gnome, icons of menu, and so on.

    People

    Jens Axboe originated this idea. Coly Li leads two intern students in Novell China R&D to work on the improvement stuffs.
