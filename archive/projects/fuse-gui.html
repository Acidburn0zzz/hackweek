    • Fuse GUI

    Tags: Idea, Desktop, SLED, Mono, FUSE

    Description

    Fuse is a way to create filesystems in userspace. One of the nice benefits of this is that non-root users can mount Fuse filesystems. Other systems exist that are similar to Fuse (KIO Slaves and Gnome-VFS) but both are only available to their respective apps, and not available through the terminal. Fuse actually mounts it's filesystems onto a local directory therefore it is accessible to **all** GUI and terminal applications.

    The main GUI would be a tray icon. From the tray icon you can see all of the current Fuse mounts, unmount any of them, or mount more.

    The GUI will only ask for the mount location and credentials and will automatically create a mount point on the Desktop for the filesystem (like OS X does). Optionally we could allow the user to override this with an "advanced" section.

    All information for mounting the filesystem will be stored in the gnome-keyring. An option to "mount on login" will be provided when the user first mounts the filesystem through the GUI.

    Another idea would be to export this functionality through Dbus so that other apps could mount / umount / query on demand (this would include shell scripts).
