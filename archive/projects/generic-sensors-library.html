    • Generic sensors library

    Tags: InProgress, Maintainability

    Description

    libsensors was designed a long time ago, and starts showing its age. Back then there was no standard kernel/user-space interface so the library had to know the specifics of every hardware monitoring driver. With Linux 2.6 and sysfs, we've defined a mostly standard kernel/user-space interface, so it should be possible to write a new libsensors that would be completely generic. Unfortunately, nobody took the time to work seriously on this. There was an attempt by students from Netherlands several months ago, but it was more like a proof of concept that is nowhere next to a releasable state.

    I can see at least 4 major advantages in having a generic libsensors: - Users wouldn't have to upgrade to the very latest version of libsensors to get support for new hardware. Upgrading the kernel would be enough. - Graphical front-ends displaying sensors value would no longer need to be updated each time a new version of libsensors is released. - The maintenance work on libsensors would drop to virtually zero, freeing resources for more interesting development work. - The library would be much smaller. Right now the library weights over 300 kB (on x86_64) and keeps growing with every new supported device.
