    • GUI driver management

    Tags: Idea, Desktop, Hardware

    Description

    The point here, is that although OpenSuSE has a great tool named YaST, user still has to solve some problems from console or 3rd party apps. I'll provide an example of my own. I have a Dell Inspiron 1525 laptop. Ubuntu 8.04.1 could manage all of my hardware either out-of-box or through Restricted Drivers Manager (project jockey). OpenSuSE couldn't configure my Broadcom 4310 wi-fi card. The official solution is to run a bash script to download a firmware. That, however, didn't work out for me. So I had to use Ndiswapper, which still couldn't be managed from within YaST. And I believe, that Broadcom issue is not the only one. So, either YaST should get extended to deal with such things (and I believe it's a better solution) or another tool should be offered - but still, all hardware issues should be able to deal with in GUI. It should also download the needed drivers from repos (or install it from installation media) when the drivers are not included by default.

    Probably a solution for it also involves another thing. Each time a user installs OpenSuSE on his box, all info concerning hardware on the machine should be uploaded to a centralized online database - like it happens in Ubuntu, Fedora (with smolt) or Mandriva (with HCL). There each entry that isn't supported out-of-box should get a script that would download necessary files and write necessary config lines to make that piece of hardware running. And then, YaST should query this database, get this script and run it.
