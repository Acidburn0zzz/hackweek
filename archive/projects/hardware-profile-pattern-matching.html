    • Hardware profile pattern matching

    FYI: this has the "Novell Only" switch enabled.

    Tags: Idea, Hardware

    Description

    Installing Linux on any type of computer is easy nowadays. Getting it to work with all the on-board or add-on devices is still hard.

    Support for special features e.g. extra keys on the keyboard, etc., almost impossible without manual configuration With a proper detection of the hardware, this should be automatable.

    The idea is to improve detection of computer systems by applying pattern matching. So instead of direct hardware detection (extra keys are probably quite hard to autodetect ...), indirect detection is applied. This should work since computers have a unique combination of chips and chip ids (//pci fingerprint//).

    With this information, the installer will be able to detect specific vendor systems and install (resp. configure) the correct device support. The database to match against is either on the install media (CD) or, in combination with online installs, on a server on the internet.

    This idea could be patentable, so I enabled the 'Novell only' flag for now.
