    • HTML Link Checking In Common Editors

    Tags: Idea, Desktop, Web20

    Description

    People typo all the time, and so every single editor I know has a spell checker some how.

    People get their HTML links wrong all the time too (such as Nat's post for this portal that fumbled the link to the Awards page) and yet while there are link checkers, they are not common, and not integrated into editors.

    The idea: take some existing link checker and bundle it so that it can be easily used in common editors. The most common editors in my life are: * Vim: the link checker should be either a command line or a pipe, e.g. from within vim ":.,$ fmt" will line-wrap your prose, use something similar to link check your text. Especially useful when composing e-mails in Mutt or such that contains links. * EMACS: see above. * Thunderbird: Has a spell checker built in, I want a link checker. * KMail: see above. * Seamonkey Composer: My favorite HTML editor, has a spell checker, should have a link checker.

    P.S. I checked the links in this idea page **manually** :-P
