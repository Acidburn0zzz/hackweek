    • Implement $PID in AppArmor Policy Language

    Tags: Idea, Security

    Description

    AppArmor profiles could be made tighter if one specify /some/path/to/$pid instead of /some/path/to/[0-9]*

    Idea: implement a special token for $pid (and for $uid) that is loaded into the AppArmor module, and checked at access time to match against the actual PID of the confined process.

    Caveat: In AA versions in 10.2 and earlier, the access control was done by string matching, and so it was relatively simple to just insert special case logic for the $pid and $uid tokens. In 10.3 and later, AA uses a compiled FSM (Finite State Machine) making this somewhat more difficult to implement. Special acceptance states would have to be built into the FSM.

    People

    Crispin Cowan originated this idea. No one is currently working on implementing this idea (it's up for grabs!).
