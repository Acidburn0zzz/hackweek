    • inode based journaling

    Tags: Done, Kernel, filesystem

    Description

    For example, on ext3/4, current journaling policy is partition based. All files within the same partition have to use the same journaling mode (writeback, ordered, journal). Sometimes, only several files in a filesystem is important to be journaled. In this case, it is not necessary to journal all the filesystem. If we can only journal the specific file(s), the performance can be improved for whole partition. I dont know whether it is feasible, just input an idea here.
