    • Laptop Access Panel

    Tags: InProgress, Mobile, LowPower, Performance, openSUSE, SLED, JudgeDesktop

    Description

    There is no central point to configure typical laptop related functions (power management, display brightness, wireless networks, wireless devices such as Bluetooth).

    • *The bad:**

    We can hack on some kind of a Laptop Access Panel. It will give users:

    • *The good:**

    - the ability to control the basic functions which are currently offered by various stand-alone applications (g-p-m, nm-applet, KPowersave, KNetworkManager and so forth) - a quick overview of the current system status.

    The application should be as dumb as possible. It should use D-Bus for interprocess communication and HAL for data gathering. It should not poll to get any information. If a D-Bus backend is missing (e.g. xrandr) this project should get the ball rolling towards a discussion with the responsible subsystem maintainer(s)/programmer(s).

    People

    Timo Hoenig originated this idea. Basic mockup of the GUI is done, interested parties please contact me to get this implemented.
