    • Live CD to create virtual machine from physical (p2v)

    Tags: Idea, Virtualization, Server

    Description

    Create a SUSE-based Live CD that helps create a virtual machine from the physical machine.

    Sample use case:

    Customer has existing server, with installed apps and data. Customer prefers to not have to re-install to virtualize the server. Customer puts p2v CD in server and reboots. YaST starts (similar experience as starting an install). YaST probes hardware (discovers # of NICs, # of drives, amount of memory, architecture, etc.) YaST prompts user for destination (NFS, SMB/CIFS, iSCSI, or perhaps local hot-plugged disk) of where to create the VM's disk image(s). The local disk(s) are copied to the destination and YaST also writes out a VM configuration which matches the physical machine.

    Some advantages this has over other other manual tools (ghost or "dd"): * VM configuration created automatically to match physical machine * Can re-configure OS automatically. Example: The virtual NICs' MACs will not match the physical NICs' MACs; this could be fixed automatically (for some OSs) * Could paravirtualize automatically (for some OSs) * Can convert/compress the disk format during copying * Don't have to physically move hard drive to new server * Independent of what OS customer is running

    YaST could use the existing vm-install code to write out the VM configuration.

    Probably would only do file-backed disk images to avoid filesystem-resizing issues (for now).

    People

    Charles Coffing originated this idea. No one is currently working on implementing this idea (it's up for grabs!).
