    • Make emacs as a better IDE to manage projects

    Tags: Idea, DevTool, Emacs, Desktop, openSUSE, SLED

    Description

    Emacs is a better editor but it does not manage the project's source like source-insight or jbuilder. Minimally it should do better source files management

    - User creates a new project [just gives a name for the project and folder which contains the source]. It should read the whole subtree generate tags and manage them.

    - One small window on left side to display all files related to the current working project

    - One window to list the members for the current buffer

    - Hide the above two window if there is no focus for some time probably.

    - Probably the cursor is focused on a function call the corresponding function definition should be shown in other window.

    - Load themes for various editor's key bindings. (high-lighting, region-commenting, jump to caller or definition or bookmarks,search file, search project, goto-line, even cut copy paste :-) ...)

    But most of the editors will make use of the tool bars. Not sure, probably we can have some common key bindings or tool bar is feasible?

    anyway we can keep on adding it :-)
