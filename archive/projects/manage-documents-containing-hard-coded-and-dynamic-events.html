    • Manage documents containing hard coded and dynamic events

    Tags: InProgress, Web20, HelpWanted, HW-Hacker, HW-Designer

    Description

    Many documents such as contracts contain events on which certain things are scheduled to happen. The most obvious examples are effective dates, expiry dates, survival periods etc. When these are hard coded in the document, it is easy to manage them in a database. However, even though most contracts have a fixed effective date to start on and expiry date to end on, there is often a provision that a contract will automatically renew itself for further periods, if not terminated before a certain date. The 'certain date' will normally be defined as something like '90 days before the expiry date'. Thus, this has to be determined. There are other 'rolling events' which have to be considered as well, such as milestones.

    Once the events have been stored in a database, the person responsible for the contract should be alerted by email (etc) on %event - x days where x is the number of days that the person responsible needs to act on the event (some contracts need to be terminated in writing, and sent by FedEx etc, so it may take some time).

    The actual documents themselves can be stored centrally, with access being only granted to the person responsible (or a group of responsible persons). It could be possible to run the scanned versions through tesseract etc in order to be able to perform a full-text search from a frontend.
