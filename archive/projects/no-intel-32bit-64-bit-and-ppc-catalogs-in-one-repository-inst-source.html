    • No intel (32bit), 64 bit and ppc catalogs in one repository (inst-source)

    Tags: Idea, yast, repository, installation, rpm

    Description

    One repository supports 3 different main architectures of CPU's and YaST download all of them - why? If user has one CPU's arch he or she do not need 2 others. Repositories should be divided into 3 directories, so "YaST Software Management" would start almost 3 times faster!

    I know that 64 bit CPU's can handle 32 bits applications, but no vice versa!===== People =====

    Sebastian Furdal originated this idea. No one is currently working on implementing this idea (it's up for grabs!).
