    • package wizard for the build service

    Tags: TryMe, InBuildService, buildservice

    Description

    The Build Service needs a wizard to generate spec files for packaging newbies (who might be experts in other areas, like programmers wanting to build their projects in the Build Service). There's already something like that [1], but it's more a "smart template" than a wizard. For the wizard to be really helpful, it needs to be able to analyze the source to find out how to compile the package (plain make, ./configure && make, cmake and so on). It's a bit tricky though to even extract arbitrary tarballs in a secure way.

    People

    Michal Marek originated this idea and is working on it.
