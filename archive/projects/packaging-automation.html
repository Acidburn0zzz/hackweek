    • Packaging automation

    Tags: Idea, Community, openSUSE, HelpWanted

    Description

    Many parts of packaging process are based on mechanical work and it is possible to develop automatic tools to perform these steps automatically:

    - Update of descriptions. - Know about different versioning schemes and find the latest version. - Source lookup and suggestion of correct packaging style (e. g. GNU configure, perl package, python package). - Watch and report build option changes. - Watch for checked files to correctly collect BuildRequires. - Suggest package splitting and collect file lists. - Generate more acurrate dependencies.

    People

    Stanislav Brabec originated this idea. No one is currently working on implementing this idea (it's up for grabs!).

    Stanislav Brabec created several incomplete implementations of this idea in bash (osbuilder, pwiz)

    Rajagopal Natarajan wrote Abstract Package Build System and Abstract Package Build Description framework in ruby as a part of Google Summer of Code project.
