    • PulseAudio support for MythTV

    Tags: Idea, openSUSE, Done, MythTV

    Lightning Talk

    Description

    http://www.pulseaudio.org has some great features. In order to output to pulseaudio from MythTV, you have to use an oss emulation wrapper (padsp). Patch myth to have real pulseaudio support.

    People

    Wade Berrier originated and is working on this idea.

    Related Materials

    I had Pulseaudio's 'simple' api implemented for a proof of concept. Obviously using this api, A/V was out of sync. After looking at the asynchronous api, it was faster to get Myth's ALSA output to work with PulseAudio. It had to be updated to not use MMAP. So the patch below checks for MMAP, and falls back to non-MMAP if it isn't available. A/V sync is perfect using this method.

    Fixing alsa was also nice in the fact that no new dependencies were needed for myth.

    Patch posted to the myth bug http://svn.mythtv.org/trac/ticket/3598 .
