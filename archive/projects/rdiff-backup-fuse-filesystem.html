    • rdiff backup FUSE filesystem

    Tags: Idea, FUSE, filesystem

    Description

    rdiff backup has many advantages over a hard link backup system like dirvish. For example, backing up large database files with rdiff just takes the space of the deltas whereas with hard link backup systems the whole file is copied.

    Hard link backup systems have one feature in general that keeps people using them; the ability to easily navigate and restore files. Since hard link backup systems just use filesystem features for their backups all backups appear on the filesystem as the tree looked at the time of the backup.

    This feature could be solved by creating a FUSE filesystem that presented the rdiff backups exactly how the hard link backups are presented. It would also be important to be able to simply copy from the FUSE filesystem in order to restore backups. It would be acceptable to have the filesystem be read only, which would actually have the added side affect of making it harder for users to accidentally erase their backups while browsing them.

    People

    Thomas Wiest originated this idea. Miklos Szeredi is interested in implementing or helping out with this idea.
