    • Run linux desktop from a ramdisk that gets persisted to disk on shutdown

    Tags: Kernel, Desktop, Performance, Idea, openSUSE, SLED, SLES

    Description

    People

    Thomas Wiest and Wade Berrier originated this idea. No one is currently working on implementing this idea (it's up for grabs!).

    Related Materials

    The idea came from seeing a $1200 ramdisk with a sata interface built in. The disk has a battery and a flash disk (or solid state) to persist data even if the power goes out.

    But, this should be possible with using plain system ram as well, which would be much cheaper. For my machine personally, I don't have power outages to worry about, and the machine is always on.

    The idea would be that you could load up a disk image into a ramdisk from a harddrive on bootup. That means bootup would be slow, but that also means that you'd never hit the disk. You could use a harddrive as additional slow storage (music, photos, etc...) if needed. But if you had 6-8GB of ram (which doesn't cost that much), you could use 2GB for memory, and then 4-6 GB for the root partition.

    You could also boot up an image from the network for a diskless machine.

    You would need to be able to have the ability to overflow to a harddrive just in case the ramdisk fills up. (Use swap memory for this?)

    Persisting the state of the machine could be done on shutdown, or daily from a cronjob at 1am. Or if on a laptop when your battery power goes below a certain point.

    There may be some kernel filesystem buffering tricks to get the same result, but they would *only* be acceptable if the disk was *never* touched after the system was up and running.

    Speaking of laptop usage, it could possibly save a lot of battery power if you were never reading/writing to a harddisk.
