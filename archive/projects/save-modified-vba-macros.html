    • save modified vba macros

    Tags: office, InProgress

    Description

    Currently with the Open Suse and Novell versions of openoffice we import vba macros and run them natively. Working on vba interop time after time I need to make minor updates to macros in excel in order to ensure the same macro works in both Excel and Openoffice. I end up modifying the document/macro in excel ( on another machine , real or virtual ), transferring it back to my linux host, opening in Openoffice, testing, returning to the excel document to make more modifications, re-importing the document etc. etc. etc. So while testing it's possible to make changes in place to the vba macros imported but you can't save those changes to the excel document you are editing, particularly this is very annoying when trying to test macros that run on document load. The idea is to be able to modify imported vba marcos and export them to excel :-) Michael Meeks did some investigation about this, produced some patches that need love and attention in order to integrate into openoffice
