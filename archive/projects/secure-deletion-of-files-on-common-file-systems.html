    • Secure deletion of files on common file systems

    Tags: Idea, Kernel, filesystems, security

    Description

    Several Linux file systems support tagging of files for secure deletion but don't actually honor them. When files are no longer used, the most common method of deletion is to remove the file metadata and mark the blocks freed. This means that while there is no longer a link to the deleted file, the file's data exists as certainly as if it weren't deleted at all. Secure deletion means that when the file is deleted, the existing file data is overwritten before it is freed. There are multiple levels of secure deletion (how many times it is overwritten), but for most purposes once is enough. There is an obvious performance impact for this, since it essentially means that the entirety of the file will be written again as if it were new before it is deleted. The flag should be available and honored at the inode and file system level.

    People

    Jeff Mahoney originated this idea. No one is currently working on implementing this idea (it's up for grabs!).
