    • Standalone download program for openSUSE images

    Tags: Idea, Community, Network, openSUSE, HelpWanted

    Description

    Downloading openSUSE images successfully is still a major exercise for many people, especially when the network connection is less than optimal. There is technology which solves all the problems nicely [[http://mirrorbrain.org|metalink generator]] on download.opensuse.org and [[http://en.wikipedia.org/wiki/Metalink|metalink clients]]. However the needed client software does not exist natively in browsers, nor is it installed on users systems by default, thus requiring extra learning steps.

    The idea is to create a smallish standalone program (which is a small download) that users install on their machine, and which would contain a metalink client and take care of the download procedure, including

    * downloading from close mirrors * parallel download from multiple mirrors * deal nicely with failing mirrors * automatically repair broken downloads * using the segment hashes that the server provides to verify integrity of the download * use the PGP signature that download.opensuse.org (can) provide(s) to verify authenticiy of the image (see https://bugzilla.novell.com/show_bug.cgi?id=381731)

    All this is nicely implemented in some metalink clients already. For instance, aria2c is an excellent client. The standalone downloader mainly has to make this available to the users in an easy way, and on multiple platforms (Linux, Windows, OSX).

    The program could also be used * to ask users questions to find out which download they need (x86_64? gnome/kde?) * to act as wubi-like client (see http://idea.opensuse.org/content/ideas/wubi-for-opensuse) * to burn the image to CD/DVD (for conveniency)

    Since downloading large images is such an obstacle, making this procedure easier has a HUGE potential to make openSUSE more popular.

    People

    Peter Poeml originated this idea. Josef Reidinger currently working on implementing this idea.
