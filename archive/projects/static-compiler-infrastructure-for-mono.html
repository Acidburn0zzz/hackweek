    • Static compiler infrastructure for Mono

    Tags: DevTool, Mono, InProgress, compiler

    Description

    Within the Mono project, we have currently a few static compilers like gmcs, mcs, vbnc or mjs. They are all tied to the version of the runtime they are using, so they are not able to emit assemblies for another version of the runtime, and they hardly share code.

    This project is meant to implement a compiler infrastructure one can use and extend to write its own compiler, so eventually, all benefit from using a single infrastructure.

    To achieve the runtime independence, the project will use Cecil, a managed assembly writer already part of the Mono platform.

    = Draft Planning

    In a first step, the idea is to extract a compiler infrastructure from the Boo codebase, to refactor it to make it more C#ish. (That will likely take the whole week)

    In a second step, we would complete the emitter that uses Cecil, to be runtime independent.

    The third step would be to port our FlowAnalysis engine to use the new infrastructure.
