    • Support both KDE and GNOME-ordered buttons in YaST

    Tags: Idea, SLED, Management, YaST

    Description

    Let YaST order the Yes/No button pairs depending on currently selected UI beckend (Gtk vs. Qt) in order to seamlessly fit not only into KDE environment, but into GNOME as well. Complex dialogs need to be able to get the information about button ordering and ordet the buttons on its own.

    People

    Jiri Srain originated this idea. No one is currently working on implementing this idea (it's up for grabs!).
