    • System wide codec registry

    Tags: InProgress, video, System, multimedia, HelpWanted, judgeMe

    Description

    SUSE Linux lacks some basic multimedia codecs like mp3, video-mpeg and others. This is mostly due to patent and licensing issues. But nevertheless the infrastructure for adding such codecs should be well prepared. We suggest to create a centralized service which

    - can be queried for installed codecs,

    - can be queried for existing codecs (repositories),

    - shall provide wrapper framework to make one codec fit into several applications

    - shall provide a default-loader plugin, that hooks into applications for API transformation.

    - shall provide a codec installer, automatically called by the default-loader plugin if necessary, for downloading existing but not installed codecs. This installation tool should allow easy 'click-yes-yes-yes' installation whenever an application encounters an unknown file format.

    The concept of pluggable codecs could also be extended beyond codecs and the multimedia area. Any converter software that allows transformation between two file-formats (be it streaming, or file based) should register to this centralized service.

    Example:

    - Convert text to graphics: call imagemagick

    - Convert graphics to text: call tesseract-ocr

    This idea also calls for the definition of a codec interface standard. Both multimedia-codecs and converters need a standard interface. A unix style command line interface may be sufficient for simple converters or codecs, but those that want to provide progress indicators or other feedback and those that want to offer parameter settings to the user need communication channels not provided by the command line shell. A standardized API (codec-vs-user, codec-vs-application) shall be defined in a GUI independent manner, allowing a codec to run in streaming, interactive, batch and other modes. How far will e.g. gstreamer bring us here?

    In the absence of more specialized interprocess communications, dbus could handle the queries to this service.

    I believe that large distributors can best establish such integrated solutions.
