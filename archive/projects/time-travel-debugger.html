    • Time travel debugger

    Tags: Idea, DevTool, bugs

    Description

    A time-travel debugger is capable of stepping //backward// through the entire execution history of a program. This will require recording enough information during the run of the program that every detail of the program state at any point during the run can be reconstructed without having to run the program again.

    Benefits:

    - No need to iteratively re-run programs should speed up debugging. Eliminate the run, crash, set breakpoint, rerun cycle. - Heisenbugs should be much easier to track down because you only need to capture them under debug once - The ability to answer the question "When was X set to this value?" efficiently in a debugging situation

    This would be a revolution in debugging.

    People

    Nat Friedman originated this idea. Robert O'Callahan has been working on this project.
