    • Use git for build service source control

    Tags: Idea, BuildService, DevTool

    Description

    For the build service, branching/merging/bisection/diff could be made really easy to use if we used git or another distributed version control system. Git would work well with how things are aligned now if each project were a repo and and new projects could be started by cloning an existing repo or subset of a repo. Merging back to factory from community submissions could be done by "cherry picking" in git or merging whole repos together.

    People

    JP Rosevear originated this idea. Michael Wolf is working on it, and would appreciate input and assistance.
