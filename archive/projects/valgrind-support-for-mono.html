    • Valgrind support for Mono

    Tags: Idea, Mono, DevTool, InProgress, TryMe, judgeUnixAttack

    Description

    Currently Valgrind can not show the actual symbols for addressed of code generated by the Mono JIT engine. The idea would be to modify valgrind to provide a client API for Mono to register the JITe d methods and get the proper stack traces.

    There is a patch currently in: http://tirania.org/tmp/valgrind-mono.patch

    A few more features that need to be implemented: * Range unregistration (unloading of appdomains) * Line number support as well.

    People

    Miguel de Icaza originated this idea. No one is currently working on implementing this idea (it's up for grabs!).
