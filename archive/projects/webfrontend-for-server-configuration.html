    • webfrontend for server configuration

    Tags: InProgress, Community, Management, Server, LAMP, openSUSE, SLES

    Description

    Create a webfrontend to configure server services easily (http, mail, mailinglist, database, ftp etc.). The system should be independent of server software running, independent of the operating system, be extendible with plugins, support multiple users (role based) and multiple hosts (consolidate services to dedicated machines).

    The authority for configuration will be the database the application works on. The configuration may be pushed to the clients or pulled from the clients. They write it to their systems config files or config services (eg. LDAP), so at this point they can even use existing configuration tools like YaST. All servers and services should remain running even if the authoritive database is down (just configuration updates will not be distributed).

    There are already solutions in place (webmin or vhcs, to name two well known) but they are mainly done from a different approach.

    People
