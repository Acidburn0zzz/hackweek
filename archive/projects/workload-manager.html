    • Workload Manager

    Tags: Idea, Kernel, Server, Performance, Management

    Description

    A good modern enterprise grade OS needs a workload manager in order to properly support different kind of concurrent payloads.

    A sysadmin running different big applications in a big multiprocessor box must be able to determine which of these applications will gain priority access to resources (cpu, memory, i/o) in case of resource constraints.

    i.e. A rule could state that the payroll application processes, or the database running a large e-commerce site, should have priority over the e-mail dispatcher.

    As a parallel benefit of a workload manager implementation, comes the possibility of having statistics on resource consumption for the different classes or sets of processes defined.

    People

    Serxio Barral originated this idea. No one is currently working on implementing this idea (it's up for grabs!).
